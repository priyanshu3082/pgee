📦 Data Structures & Algorithms
Highest weightage — ~25% of CSE paper
▼
  • Arrays and multi-dimensional arrays
  • Linked Lists (singly, doubly, circular)
  • Stacks and Queues (array & linked)
  • Trees: Binary, BST, AVL, B-trees
  • Heaps and Priority Queues
  • Hashing: Collision resolution strategies
  • Graphs: Representation (adj matrix/list)
  • BFS, DFS, Dijkstra's, Bellman-Ford
  • Floyd-Warshall, Prim's, Kruskal's
  • Sorting: Merge, Quick, Heap, Counting, Radix
  • Searching: Binary search, interpolation
  • Recursion and backtracking
  • Dynamic Programming (LCS, LIS, Knapsack)
  • Greedy algorithms
  • Divide and conquer
  • Amortized analysis
  • Time & Space complexity (Big-O, Ω, Θ)
  • NP-Completeness basics
💡 Key Focus: Pointer-based DS in C, memory management questions, and DP problems are most frequently tested per past experiences. Trees + graphs dominate the paper.
🖥️ Operating Systems
High weightage — ~18% of CSE paper
▼
  • Process management, states, PCB
  • CPU scheduling algorithms (FCFS, SJF, RR, Priority)
  • Process synchronization: semaphores, mutexes
  • Deadlock: conditions, prevention, avoidance, detection
  • Banker's algorithm
  • Memory management: paging, segmentation
  • Virtual memory, page replacement (FIFO, LRU, Optimal)
  • Thrashing and working sets
  • File system: FAT, inode, directory structure
  • Disk scheduling (SSTF, SCAN, C-SCAN)
  • I/O management, device drivers
  • Inter-process communication (IPC)
  • Threads and multithreading
  • Critical section problem, Peterson's solution
💡 Key Focus: Numerical questions on page replacement algorithms (LRU page faults), Banker's algorithm, and scheduling Gantt charts appear regularly.
🗄️ DBMS
High weightage — ~15% of CSE paper
▼

  • SQL: SELECT, JOIN, GROUP BY, HAVING, subqueries

C — Classic Pointer-based Linked List (exam favorite)
// PGEE often tests pointer-based DS in C
struct Node {
    int data;
    struct Node* next;
};

// Insert at head — O(1)
struct Node* insertHead(struct Node* head, int val) {
    struct Node* node = (struct Node*) malloc(sizeof(struct Node));
    node->data = val;
    node->next = head;
    return node;
}

// Reverse a linked list — classic PGEE question!
struct Node* reverse(struct Node* head) {
    struct Node *prev = NULL, *curr = head, *next;
    while (curr) {
        next = curr->next;
        curr->next = prev;
        prev = curr;
        curr = next;
    }
    return prev;
}

🔑 Deadlock — Must Know
  • 4 Conditions (all required simultaneously): Mutual exclusion, Hold & wait, No preemption, Circular wait
  • Prevention: Negate any one condition
  • Avoidance: Banker's algorithm — check safe state before granting
  • Detection: Resource allocation graph (RAG), cycle = deadlock for single instances
  • Recovery: Kill process, preempt resources, rollback

📡 TCP Congestion Control — Phases
  • Slow Start: cwnd doubles each RTT (exponential) until ssthresh
  • Congestion Avoidance: cwnd increases by 1 per RTT (linear) after ssthresh
  • Fast Retransmit: 3 duplicate ACKs → immediate retransmit (no timeout wait)
  • Fast Recovery: ssthresh = cwnd/2; cwnd = ssthresh (skip slow start)
  • On timeout: ssthresh = cwnd/2; cwnd = 1; back to slow start

Digital Logic — K-Map & Flip-Flops

🗺️ K-Map Simplification Rules
  • Group cells with 1s in powers of 2 (1, 2, 4, 8, 16...)
  • Groups must be rectangular; wrapping around edges is allowed
  • Larger groups = simpler expression; always aim for largest possible group
  • Each variable eliminated per doubling group size
  • For SOP: group 1s; For POS: group 0s (max-terms)
Flip-FlopInputsOperationCharacteristic Equation
SRS, RSet/Reset (S=R=1 invalid)Q(t+1) = S + R'Q
JKJ, KSet/Reset/Toggle (J=K=1 → toggle)Q(t+1) = JQ' + K'Q
DDData/Delay (Q follows D)Q(t+1) = D
TTToggle (T=1 → toggle; T=0 → hold)Q(t+1) = T ⊕ Q

Quick Revision

Last-Minute Revision Sheet

Critical facts, formulas, and mnemonics to review in the final hours before the exam.

⚡ Must-Memorize Facts
  • AVL tree rotations: LL → Right rotation; RR → Left rotation; LR → Left then Right; RL → Right then Left
  • Heap insert/delete: Both O(log n). Build heap: O(n), not O(n log n)!
  • Deadlock 4 conditions: Mutual Exclusion, Hold & Wait, No Preemption, Circular Wait
  • ACID properties: Atomicity, Consistency, Isolation, Durability
  • TCP 3-way handshake: SYN → SYN-ACK → ACK
  • IPv4: 32-bit; IPv6: 128-bit. Subnet mask /24 = 256 addresses, 254 usable hosts
  • Process vs Thread: Threads share code/data/heap but have own stack/registers
  • Semaphore vs Mutex: Mutex is binary semaphore with ownership; semaphore allows counting
  • B-tree order m: Every node has at most m children; at least ⌈m/2⌉ children (except root)
  • NP-complete examples: SAT, 3-SAT, Traveling Salesman, Vertex Cover, Clique, Knapsack (decision)

Exam Day

Exam Strategy

Paper I: General Aptitude (2:00 – 3:30 PM)
Start with questions you're confident about. Skip and come back to tricky ones. Don't spend more than 2 min per question. With 25% negative marking, only attempt if ≥60% confident. Target: attempt 70–80% of questions.
Paper II: CSE Subject (3:30 – 5:00 PM)
Start with your strongest subjects (DS/Algorithms or OS). For subjective questions, show all steps — partial credit may be awarded. Manage time: 10–12 min per subjective question.
Time Allocation in Paper II
DS & Algorithms: 25 min | OS: 18 min | DBMS: 18 min | Networks: 12 min | Digital Logic: 10 min | C Programming: 10 min | Discrete Maths: 10 min | Review: 7 min
The 25% Negative Marking Rule
Only guess if you can eliminate at least 2 options out of 4. If you're completely unsure, leave it blank. Attempted wrong = −0.25 marks; blank = 0. A wrong guess hurts more than a skip.
